一、核心内容
1.自定义Git
输出着色：git config --global color.ui true，让命令输出有颜色，更易读。
配置别名：将长命令简化为短命令，大幅提升输入效率。
配置文件：别名配置保存在用户主目录的 .gitconfig文件中，可直接编辑。
2.忽略特殊文件 
作用：让Git忽略指定的、不需要纳入版本管理的文件。
编写规则：在仓库根目录创建 .gitignore文件，每行一个模式。
忽略所有 .class文件：*.class
忽略所有隐藏文件：.*
例外规则：在模式前加 !，如 !.gitignore表示不忽略 .gitignore文件本身。
重要原则：.gitignore文件本身需要提交到版本库进行管理，确保所有协作者共享同一套忽略规则。
3.SourceTree
作用：提供图形界面操作Git，可视化分支、提交历史，简化部分操作。
本质：本质上仍是调用Git命令。操作失败时，仍需关注其返回的Git错误信息来排查问题。
定位：是命令行操作的补充和增强，而非替代。理解底层命令仍是基础。

二、思考
整个git的学习让我意识到，Git不仅是一个工具，更是一种高效、安全的工作方式。从手动备份到通过分支开发协作，从复杂命令到用别名一键操作，它很好地解决了协作和版本管理的核心问题。

三、demo
# 1. 启用输出颜色，让命令结果更易读
git config --global color.ui true

# 2. 配置常用别名
git config --global alias.st status     
git config --global alias.co checkout   
git config --global alias.ci commit      
git config --global alias.br branch     
git config --global alias.last 'log -1'  

# 3. 查看所有配置
git config --global --list

# 4. 创建并编辑 .gitignore 文件
touch .gitignore
*.class
.*
!.gitignore
/target/
/node_modules/
*.log

# 5. 将 .gitignore 文件加入版本管理
git add .gitignore
git commit -m "add .gitignore file"

# 6. 使用别名进行日常操作
git st        
git co -b new-feature  
git add .
git ci -m "update" 
git br
