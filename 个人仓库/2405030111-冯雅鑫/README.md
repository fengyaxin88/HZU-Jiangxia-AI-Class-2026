一、Python 面向对象 & 异常处理学习思考
学习问题：初期混淆父类子类的继承逻辑，不知道如何调用父类初始化方法，解决：通过super().__init__()快速继承父类属性，简化子类代码，在 AI 学习类的扩展中能快速复用基础属性和方法。
异常处理关键：Tensor 数据处理中易出现维度不匹配、类型错误、除零错误，用try-except精准捕获各类异常，能避免程序直接崩溃，完美适配 PyTorch 的张量运算场景，提升代码鲁棒性。
面向对象优势：将 AI 学习的核心属性和行为封装成独立的类，后续扩展深度学习、大模型相关功能模块时，可通过继承快速实现功能迭代，大幅提高代码的复用性和可维护性。

二、PyTorch Tensor 核心学习思考
核心问题：创建 Tensor 时未手动指定数据类型，导致后续矩阵运算、维度变换出现类型不兼容报错，解决：根据实际运算需求，用torch.tensor()/torch.FloatTensor()明确指定数据类型，统一所有运算的 Tensor 类型。
维度变换技巧：reshape/squeeze/unsqueeze是 Tensor 操作的基础核心，处理图像、矩阵等深度学习数据时，需灵活调整维度以匹配网络模型的输入要求，尤其要注意升维 / 降维后的维度数是否符合运算逻辑。
自动求导理解：requires_grad=True是开启 PyTorch 自动求导的关键开关，反向传播执行backward()后，梯度值会自动保存在张量的grad属性中，这是深度学习中梯度下降法训练权重的核心原理，所有模型的参数更新均基于此实现。
Git 使用问题：首次提交代码时忘记配置全局的user.name和user.email，导致仓库提交失败，解决：通过git config --global user.name "冯雅鑫"和git config --global user.email "xxx@xxx.com"配置全局信息后，顺利完成仓库克隆、本地提交和远程推送。
